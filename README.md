# めんどりさん
面談の面倒な段取りをコッコッと済ませてくれる、個人面談スケジューリングアプリ

## 📌 概要

めんどりさんは、学校の個人面談スケジューリング作業を自動化する
Google Apps Script製のWebアプリです。

希望日程を保護者にヒアリングするためのGoogleフォームの生成、
フォームの回答の読み込み、自動スケジューリングと予定表の出力までを行います。
すべてのデータはGoogleスプレッドシート内に保存され、
外部サービスへのデータ送信はありません。

対象ユーザー：小〜中規模校の先生  
動作環境　　：PCブラウザ（Googleアカウントでログイン）

## 🖥️ 画面構成

4つの画面で構成され、順番に進む設計です。

| 画面 | 名前 | 内容 |
|------|------|------|
| 🥚 画面① | 設定・フォーム生成 | 面談日・時間帯・1コマ時間を設定し、保護者向けGoogleフォームを自動生成する |
| 🐣 画面② | 回答データ確認 | フォームの回答を読み込み、各生徒の対応方針を確認・設定する |
| 🐤 画面③ | スケジューリング | 手動指定の日時を設定し、自動スケジューリングを実行する |
| 🐔 画面④ | 結果確認・出力 | スケジューリング結果をカレンダーグリッドで確認し、Googleスプレッドシートに出力する |

## ⚙️ 技術構成

| 項目 | 内容 |
|------|------|
| 言語 | Google Apps Script（GAS）/ HTML / CSS / JavaScript |

## 🚀 セットアップ

詳しい手順は [セットアップ手順書](docs/セットアップ手順書.pdf) をご覧ください。

### 必要なもの
- Googleアカウント（学校で使用しているもの）
- 生徒の名簿データ（学年・組・番号・姓・名の5列）

### セットアップの流れ
1. Googleスプレッドシートを新規作成する
2. Apps Scriptエディタを開き、コードファイルを貼り付ける
3. appsscript.json に権限スコープを追加する
4. ウェブアプリとしてデプロイする
5. テストデータで動作確認する

所要時間の目安：15〜30分（動作確認テスト含む）

### ⚠️ テストデータについて

test/insertTestData.gs にテスト用の回答データが含まれています。

テストデータの設定は以下のとおりです。
動作確認時は画面①でこの設定に合わせてフォームを作成してください。

```
面談日：6月15日（月）・16日（火）・18日（木）・19日（金）・22日（月）
時間帯：15:30〜17:00（月・火・月）／ 13:30〜16:30（木・金）
1コマ：15分
対象　：6学年×2クラス×25名（計300名）
```

上記の面談日が過去になった場合は、insertTestData.gs 内の日付を
未来の日付に修正してから使用してください。

⚠️ 本番運用前に必ず insertTestData.gs を削除してください。
